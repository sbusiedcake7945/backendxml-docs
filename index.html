The Engine Architecture

The BackendXML ecosystem follows a Tri-Phase Pipeline to ensure maximum safety and speed.

1. Stream Lexing

Raw XML is shredded into logical tokens, identifying attributes, content, and nested children.

2. AST Mapping

The Parser validates the token stream to build an Abstract Syntax Tree, checking for language integrity.

3. Concurrent Exec

The Interpreter traverses the AST, spawning Goroutines for HTTP handlers and background tasks.

Technical API: Elements

Tag Context Capabilities
<backend> Root The global application scope. Parent of all nodes.
<import> Global Imports .backend files or recursively loads entire directories.
<function> Handler Native code gateways. Executes raw Go, JS, or HTML.
<var> Hybrid Dual-scope variables (Global/Request). Supports {{var}} injection.
<route> Global Static and Dynamic HTTP path routing. Thread-safe execution.
<database> Global Unified data gateway. Supports SQLite, JSON, or custom extension types.
<db.get> Handler Fetches data from the connected database directly. E.g. <db.get q="SELECT..." />.
<query> Handler Executes database commands. Supports <query q="..." type="..."> notation.
<if> Handler Conditional execution using JS-style expressions. Supports <elseif> and <else>.
<get> Hybrid Injects request metadata (ip, url). Security-hardened against DB injection.
<include> Global C-style inline file inclusion with detailed includeError tracing.
<open> Global Asynchronous file execution in an isolated goroutine.
<call> / Custom Handler Executes named functions. Supports isolated parameter passing.

Polyglot Systems & Multi-Language Execution

BackendXML breaks language silos, allowing you to inject native performance where it matters most. It supports native executions of Go, Python, Node.js (Javascript), and C/C++ natively and concurrently.

Parameter Isolation & Variable Resolution: When calling a polyglot function (e.g. <my_func name="John" />), variables are isolated and mapped. Variables in both inline code block and external scripts (loaded via src="...") are dynamically resolved using the standard {{name}} placeholder!

Go Native Block

<function type="go">
  package main
  import "fmt"
  func main() {
      fmt.Println("Concurrency Active")
  }
</function>

Python Code Block & External Scripts

Runs using python3 (falls back to python). Inline constraints are ignored due to raw block parsing!

<function type="python" src="./scripts/hello.py" />
<!-- Or Inline: -->
<function type="python">
  print("Hello from Python!")
  print(1 < 2) # Raw block ignores XML validations!
</function>

C/C++ Dynamic Compilation

Compiles inline code or external files via gcc/g++ to a safe temporary binary, runs it, and discards it automatically.

<function type="cpp" src="engine.cpp" />
<!-- Or Inline: -->
<function type="c">
  #include <stdio.h>
  int main() {
      printf("Compiled and Executed dynamically!");
      return 0;
  }
</function>

Node.js / JS Engine

Runs directly in node. Supports full npm library scopes available on the machine.

<function type="nodejs">
  const os = require('os');
  console.log("Memory Available:", os.freemem());
</function>

Unified Data Layer & Dynamic Extensions

One tag to rule them all. Switch between SQL, JSON, or registered custom extensions dynamically using a unified API.

Dynamic DB Driver Registry: The engine provides Go interfaces to register custom database drivers. Our default custom driver operates as a file-based log and text database connection out-of-the-box!

Database Connections

Specify connection details and driver types dynamically.

<database connection="file.db" type="sqlite" />
<!-- Custom DB extension: -->
<database connection="plugin.log" type="custom" />

Executing & Fetching Directly

Use <db.get> to render query outputs directly inside views or store results in local variables.

<!-- Fetch and print directly: -->
<p>Users list: <db.get q="SELECT name FROM users" /></p>

<!-- Execute commands: -->
<query q="DELETE FROM logs" type="sqlite" />
Pro Tip: All database paths support ./ relative syntax for maximum portability.

Conditionals: Pure Intelligence

Control the flow of your application with high-level logic blocks.

<if condition="'{{user_id}}' == '7945'">
  <print>Welcome, Developer!</print>
<elseif condition="{{age}} >= 18">
  <print>Access Granted.</print>
<else>
  <print>Access Denied.</print>
</if>

Metadata: Environmental Awareness

Inject real-time request data directly into your views or logic.

Available data points:

  • <get data="ip" /> - Resolves to the client's IP address.
  • <get data="url" /> - Resolves to the full requested path.
Security Alert: Metadata tags are automatically sanitized and blocked within <query> blocks to prevent data leaks.

Modularity & Block Variables

Split your application into manageable components and leverage multi-line variables natively.

<include file="..." />

Seamlessly merges external logic into the current scope. If an error occurs, the engine provides an includeError trace pointing to the exact line in the included file.

<var name="..."> Block Definitions

Variables can now be defined as block structures containing multi-line content, HTML tags, or nested placeholders. Perfect for template layout wrapping!

<var name="dashboard_widget">
  <div class="card">
      <h3>Title: {{widget_name}}</h3>
      <p>Active connections resolved.</p>
  </div>
</var>

DSL & Custom Tags

Elevate your code by using named functions as native XML elements.

<!-- Definition -->
<function name="alert" type="html">
  <div class="alert-{{type}}">{{message}}</div>
</function>

<!-- Usage -->
<alert type="error" message="Critical Failure!" />

Advanced Mastery: Encryption

Handling secure data is a first-class citizen in BackendXML using Go native blocks.

<route path="/encrypt">
  <handler>
    <var name="secret" value="7945_CODE_PLATINUM" />
    <function type="go">
      // Example: Base64 Obfuscation Logic
      encoded := base64.StdEncoding.EncodeToString([]byte("{{secret}}"))
      fmt.Fprint(w, encoded)
    </function>
  </handler>
</route>
Developer Tip: Use the <function type="go"> block to access the full crypto/ package from the Go standard library for AES or RSA implementations.
